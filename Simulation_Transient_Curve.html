Session Objective

Welcome to this tutorial session! We will explore one of the most common, powerful, yet elegant astrophysical techniques to detect exoplanets around their host stars — the transit method.

"In astronomy, you often learn a lot from what you can't see… especially when it causes a tiny but regular blip in what you can." 😉

In this session we explore:

  • What light curves are and how to make sense of them
  • How to detect tiny dips caused by planetary transits
  • How to model these dips using Python tools like batman
  • How to find planet and orbital properties through this

PLD Systematics Correction

Pixel Level Decorrelation (PLD) removes instrumental systematics from the raw photometry. It uses a PCA-based approach to separate astrophysical signal from spacecraft-induced trends.

python — aperture + PLD correction 
from lightkurve.correctors import PLDCorrector

# Create aperture mask (threshold = 3σ)
aperture     = tpf.create_threshold_mask(threshold=3)
raw_lc       = tpf.to_lightcurve(aperture_mask=aperture)

# Apply PLD correction
corrector    = PLDCorrector(tpf)
corrected_lc = corrector.correct(aperture_mask=aperture)

# Plot raw vs corrected
plt.plot(raw_time[q], raw_flux[q] / np.nanmedian(raw_flux[q]) + 0.06, 'k', label="Raw")
plt.plot(corr_time[m], corr_flux[m] / np.nanmedian(corr_flux[m]), 'g', label="PLD Corrected")

Box Least Squares — Period Finding

The Box Least Squares (BLS) method is designed specifically to find periodic, box-shaped decreases in brightness — exactly the shape of a planetary transit: flat → sudden drop → flat → return to baseline.

See the AstroPy BLS documentation for full details.

Algorithm

  1. Build a log-spaced period grid from 0.1 to 15 days
  2. Compute BLS power at each period trial
  3. Locate the highest power peak → best period
  4. Reject harmonic aliases (P/2, 2P, P/3, …)
  5. Phase-fold data at the best period and plot the transit
python — BLS periodogram 
from astropy.timeseries import BoxLeastSquares

# Log-spaced period grid: 0.1 → 15 days
period_grid = np.exp(np.linspace(np.log(0.1), np.log(15), 50000))

bls       = BoxLeastSquares(time, flux)
bls_power = bls.power(period_grid, 0.1, oversample=20)

# Best peak
index      = np.argmax(bls_power.power)
bls_period = bls_power.period[index]
bls_t0     = bls_power.transit_time[index]
bls_depth  = bls_power.depth[index]

print(f"Best Period: {bls_period:.4f} days")
python — alias rejection 
# Reject harmonics: P/2, 2P, P/3, 3P, 3/2P, 2/3P
period_window_rtol = 0.01

for i in s_indices:
    is_alias = any([
        np.isclose(current_period, bls_period_1 * r, rtol=period_window_rtol)
        for r in [1, 0.5, 2, 1/3, 3, 1.5, 2/3]
    ])
    if not is_alias:
        bls_period_2 = current_period
        break

batman — Transit Modelling

batman (Bad-Ass Transit Model cAlculatioNs) generates precise analytic transit light curves for a given set of physical parameters. We first generate synthetic light curves to understand parameter effects, then fit to real TESS data.

python — generate artificial transit 
import batman

times  = np.linspace(0.6, 1.4, 100)
params = batman.TransitParams()

params.t0       = 1.0        # mid-transit time (days)
params.per      = 2.1        # orbital period (days)
params.rp       = 0.2        # Rp / R★
params.a        = 3.2        # a / R★
params.inc      = np.degrees(0.45 * np.pi)
params.ecc      = 0.0
params.w        = 0.0
params.u        = [0.1, 0.0]
params.limb_dark = "quadratic"

m    = batman.TransitModel(params, times)
flux = m.light_curve(params)

plt.plot(times, flux, "r-")
plt.title("Artificial Transit — batman")

MCMC — Posterior Sampling

Markov Chain Monte Carlo is a statistical technique used to sample the probability distribution of model parameters. Instead of giving you just one best-fit value, MCMC explores the entire space of possible solutions that fit the data — giving us uncertainties too.

We use emcee (the MCMC Hammer) with 32 walkers and 3000 steps, discarding the first 500 as burn-in.

python — emcee sampler 
import emcee, corner

ndim, nwalkers = 4, 32
p0 = initial + 1e-4 * np.random.randn(nwalkers, ndim)

sampler = emcee.EnsembleSampler(nwalkers, ndim, log_prob,
                                  args=(t, f, sigma))
sampler.run_mcmc(p0, 3000, progress=True)

samples = sampler.get_chain(discard=500, thin=10, flat=True)
k_m, a_m, inc_m, t0_m = np.median(samples, axis=0)

# Corner plot
corner.corner(samples, labels=["k", "a/R★", "inc", "t0"],
              truths=[k_m, a_m, inc_m, t0_m])

Problem Statement

The following code generates a simulated transit light curve with random physical parameters. Your task is to recover everything you can from the noisy data.

🔬 Challenge: extract period, transit depth, duration, Rp/R★, a/R★, and inclination from the synthetic light curve below.
python — synthetic challenge data 
from pytransit import QuadraticModel
np.random.seed(42)  # for reproducibility

# ── Random physically valid parameters ──
k   = np.random.uniform(0.05, 0.15)          # Rp/R★
ldc = list(np.random.uniform(0.1, 0.4, size=2)) # limb darkening
t0  = 0.0                                    # reference transit time
p   = np.random.uniform(2.0, 5.0)             # period (days)
a   = np.random.uniform(5.0, 15.0)            # a / R★

# Ensure inclination produces a transit: cos(i) < (1 + k) / a
cosi_max = (1 + k) / a
i = np.arccos(np.random.uniform(0, cosi_max))

# ── Generate 5–8 transit cycles ──
n_cycles = np.random.uniform(5, 8)
times    = np.linspace(t0 - 0.5*p, t0 + n_cycles*p, 1000)

# ── Model + noise ──
tm = QuadraticModel()
tm.set_data(times)
true_flux  = tm.evaluate(k, ldc, t0, p, a, i, 0.0, 0.0)
noise_std  = np.random.uniform(0.0005, 0.0012)
noisy_flux = true_flux + np.random.normal(0, noise_std, size=true_flux.shape)
seed=42 BLS → period batman → Rp/R★ emcee → posteriors
